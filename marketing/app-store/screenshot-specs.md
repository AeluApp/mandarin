# App Store Screenshot Specifications

## Design System

**Background:** #F2EBE0 (warm stone)
**Accent text:** #946070 (dusty rose)
**Body text:** #2A3650 (warm navy)
**Headline font:** Cormorant Garamond, semibold
**Body font:** Source Sans 3
**Hanzi font:** Noto Serif SC

**Frame style:** Minimal device frame (thin, matching #2A3650). No drop shadows. No gradients. Generous padding between frame and edge.

**Layout:** Each screenshot has three zones:
1. **Top zone** (20% height): Headline text, centered
2. **Middle zone** (65% height): Device frame with app screen
3. **Bottom zone** (15% height): Optional one-line supporting text

## Screenshot Sequence (8 Screenshots)

### Screenshot 1: Hero / Value Proposition

**Headline:** Study Chinese honestly.
**Device screen:** The main dashboard in its default state — warm stone background, today's session summary visible, the 漫 character subtly present. No notifications, no badge counts, no urgency indicators.
**Supporting text:** No streaks. No gimmicks. Just the language.

**Purpose:** Establish tone and differentiation immediately. The absence of gamification elements IS the message.

**iPhone 6.7":** 1290 x 2796. Headline at 72pt Cormorant Garamond in #946070. Device frame centered with 80px side margins.
**iPhone 6.5":** 1284 x 2778. Same layout, proportionally adjusted.
**iPad 12.9":** 2048 x 2732. Landscape-oriented dashboard layout. Headline at 96pt. Device frame with 120px side margins.
**Android:** 1080 x 1920. Headline at 60pt. Device frame with 60px side margins.

## Production Notes

- All screenshots must use real or realistic data. Never show empty states or placeholder content.
- Device frames should be current-generation models (iPhone 15 series, Pixel 8, iPad Pro 6th gen).
- Do not overlay badges, starburst graphics, or "Editor's Choice" style callouts.
- Each screenshot should be self-contained — a user scrolling quickly should get value from any single image.
- Test screenshots at thumbnail size (the size they appear in search results). Headlines must be legible at small sizes.
- For iPad, consider landscape orientation for screenshots 2, 4, and 5 where data density benefits from width.
- Export at 2x resolution for retina clarity. Final files as PNG, sRGB color space.
- File naming: `{platform}_{number}_{name}.png` (e.g., `iphone67_01_hero.png`)
