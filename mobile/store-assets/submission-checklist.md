## Apple App Store (iOS)

### Prerequisites (Manual)
- [ ] Apple Developer Program enrollment ($99/year) — https://developer.apple.com/programs/
- [ ] App Store Connect account set up — https://appstoreconnect.apple.com
- [ ] App ID registered: `com.aelu.app`
- [ ] Provisioning profile created (Distribution)
- [ ] Distribution signing certificate created in Xcode
- [ ] In-app purchase products configured in App Store Connect:
  - [ ] Pro Monthly ($14.99/mo auto-renewable subscription)
  - [ ] Pro Annual ($149/yr auto-renewable subscription)
  - [ ] Subscription group: "Aelu Pro"

### Build & Sign
- [ ] Set version number and build number in Xcode
- [ ] Minimum deployment target: iOS 16.0+
- [ ] Archive build: Xcode > Product > Archive
- [ ] Code signing: distribution certificate + provisioning profile
- [ ] Validate archive in Xcode Organizer (catches common issues)
- [ ] Upload to App Store Connect via Xcode or Transporter

### App Store Connect Configuration
- [ ] App name: "Aelu"
- [ ] Subtitle: "Patient Chinese Study" (from `ios/subtitle.txt`)
- [ ] Primary category: Education
- [ ] Secondary category: Reference
- [ ] Age rating: 4+ (no objectionable content)
- [ ] Pricing: Free (with in-app subscription)
- [ ] Availability: all territories (or select specific)

### Screenshots (Required)
Sizes needed for current App Store requirements:
- [ ] 6.7" display (1290x2796) — iPhone 15 Pro Max / 16 Pro Max (required)
- [ ] 6.5" display (1242x2688) — iPhone 15 Plus / 14 Plus (required)
- [ ] 6.1" display (1170x2532) — iPhone 15 / 14 (optional, can reuse 6.5")
- [ ] iPad Pro 12.9" (2048x2732) — required if supporting iPad

Minimum 3 screenshots per size, maximum 10. Recommended set (5 screens):
1. Dashboard — "Where you stand, honestly."
2. Drill Session — "27 ways to practice what you missed."
3. Progress Report — "Per-skill HSK readiness. No blended averages."
4. Tone Grading — "Your tones, measured honestly."
5. Settings — "Your study, your way."

Generate placeholders: `python scripts/generate_screenshots.py`

- [ ] App preview video (optional, 15-30 seconds, no voiceover required)

### Required Assets
- [~] App icon: 1024x1024 PNG (no alpha, no rounded corners) — generated by `generate-assets.sh`
- [ ] Verify icon renders well at small sizes (29x29, 40x40, 60x60)

### Metadata (Ready)
- [x] Description — `mobile/store-assets/ios/description.txt` (paste into App Store Connect)
- [x] Keywords — `mobile/store-assets/ios/keywords.txt` (99 chars, under 100 limit)
- [x] Subtitle — `mobile/store-assets/ios/subtitle.txt` (21 chars, under 30 limit)
- [x] Promotional text — `mobile/store-assets/ios/promotional_text.txt` (168 chars, under 170 limit)
- [x] What's New — `mobile/store-assets/ios/whats_new.txt`
- [ ] Support URL: https://aelu.app/support (must be live before submission)
- [ ] Marketing URL: https://aelu.app (must be live before submission)
- [ ] Privacy policy URL: https://aelu.app/privacy (required, must be live)

### Privacy Declarations (App Store Connect)
- [ ] App Privacy questionnaire completed:
  - Data collected: email address (account), usage data (drill results, session logs)
  - Purpose: app functionality
  - Data not linked to identity: analytics (session duration, accuracy aggregates)
  - Data not sold to third parties
- [ ] NSMicrophoneUsageDescription in Info.plist: "Aelu uses the microphone to record your pronunciation for tone accuracy grading."
- [ ] Push notification entitlement (if using push notifications)

### TestFlight (Before Public Submission)
- [ ] Upload build via Xcode or Transporter
- [ ] Internal testing: add team members
- [ ] External testing: requires Beta App Review (1-2 day turnaround)
- [ ] Test on physical devices:
  - [ ] iPhone (any model iOS 16+)
  - [ ] iPad (if supporting)
- [ ] Verify complete user flow: signup > dashboard > drill session > progress > settings
- [ ] Verify in-app purchases work in sandbox environment
- [ ] Verify microphone permission prompt appears correctly
- [ ] Verify dark mode renders correctly
- [ ] Verify audio playback (TTS, drill audio)

### Submission
- [ ] Submit for App Review
- [ ] Expected review time: 1-3 days (typical)
- [ ] Common rejection reasons to pre-check:
  - [ ] Privacy policy URL is accessible
  - [ ] All subscription details visible before purchase
  - [ ] "Restore Purchases" button present
  - [ ] No placeholder content in screenshots
  - [ ] App functions without network (or shows clear offline message)
- [ ] Set release preference: manual release (recommended for v1) or automatic
- [ ] Respond to any review feedback within 14 days

## Google Play Store (Android)

### Prerequisites (Manual)
- [ ] Google Play Console account ($25 one-time) — https://play.google.com/console
- [ ] App created in Play Console
- [ ] Play App Signing enabled (recommended; Google manages your signing key)
- [ ] Upload key generated and registered

### Build
- [ ] Set versionCode and versionName in build.gradle
- [ ] Min SDK: API 24 (Android 7.0)
- [ ] Target SDK: API 34 (Android 14) — required by Google Play policy
- [ ] Generate signed AAB (Android App Bundle):
  - Android Studio > Build > Generate Signed Bundle
  - Use upload keystore (not the app signing key if using Play App Signing)
- [ ] Run lint checks: no critical warnings
- [ ] ProGuard / R8 minification enabled for release

### Play Console Configuration
- [ ] App name: "Aelu - Patient Chinese Study"
- [ ] Default language: English (United States)
- [ ] Category: Education
- [ ] Tags (up to 5):
  1. Chinese language learning
  2. HSK preparation
  3. Spaced repetition
  4. Vocabulary builder
  5. Mandarin study

### Screenshots (Required)
- [ ] Phone screenshots (1080x1920 minimum): 2-8 required
  - Same 5 screens as iOS, adapted to Android dimensions
  - Generate placeholders: `python scripts/generate_screenshots.py`
- [ ] Feature graphic (1024x500) — required, most visible asset on Play Store
  - Generated as placeholder by screenshot script
  - Replace with final branded graphic
- [ ] 7" tablet (1200x1920) — required if supporting tablets
- [ ] 10" tablet (1920x1200) — required if supporting tablets

### Metadata (Ready)
- [x] Short description — `mobile/store-assets/android/short_description.txt` (78 chars, under 80 limit)
- [x] Full description — `mobile/store-assets/android/full_description.txt` (HTML-formatted for Play Store)
- [x] What's New — `mobile/store-assets/android/whats_new.txt`

### Required Assets
- [~] App icon: 512x512 PNG (32-bit, no alpha) — generated by `generate-assets.sh`
- [ ] Feature graphic: 1024x500 PNG or JPG — replace placeholder with final version

### Content Rating
- [ ] Complete IARC rating questionnaire in Play Console
  - No violence, no sexual content, no gambling, no drugs
  - User-generated content: no (all content curated)
  - Expected rating: Everyone / PEGI 3

### Privacy & Compliance
- [ ] Privacy policy URL set in Play Console (required): https://aelu.app/privacy
- [ ] Data safety section completed:
  - Data collected: email (account management), app interactions (drill results)
  - Data shared with third parties: none
  - Data encrypted in transit: yes
  - Users can request data deletion: yes
  - Data retention: until account deletion
- [ ] Ads declaration: app does not contain ads
- [ ] Target audience and content: General audience (not designed for children under 13)
- [ ] Permissions declared and justified:
  - INTERNET: required for account sync and content delivery
  - RECORD_AUDIO: pronunciation grading (tone analysis)

### Testing Tracks
- [ ] Internal testing track: upload AAB, add test accounts
- [ ] Closed testing (alpha): invite 5-10 external testers
- [ ] Open testing (beta): optional broader beta before production
- [ ] Test on multiple devices:
  - [ ] Small phone (e.g., Pixel 4a)
  - [ ] Large phone (e.g., Pixel 8 Pro)
  - [ ] Tablet (if supporting)
- [ ] Verify in-app purchases with Google Play test accounts

### Release
- [ ] Production release
- [ ] Staged rollout recommended: start at 10%, increase over 7 days
- [ ] Monitor Android vitals in Play Console (crash rate, ANR rate)
- [ ] Respond to any policy violations within 7 days
